auto-cpufreq offers flexible configuration options to customize CPU frequency scaling, power management, and battery optimization for your system.

## Configuration Methods

You can configure auto-cpufreq through:

1. **Configuration file** - Define persistent settings for charger and battery modes
2. **Command-line flags** - Override behavior temporarily or permanently
3. **Runtime options** - Control governor and turbo mode behavior

## Config File Locations

auto-cpufreq searches for configuration files in the following priority order:

<Steps>
  <Step title="Command line argument">
    Use `--config` flag to specify a custom config file path:
    ```bash
    auto-cpufreq --config /path/to/custom.conf
    ```
  </Step>
  
  <Step title="User config file">
    `~/.config/auto-cpufreq/auto-cpufreq.conf` or `$XDG_CONFIG_HOME/auto-cpufreq/auto-cpufreq.conf`
  </Step>
  
  <Step title="System config file">
    `/etc/auto-cpufreq.conf` (default location)
  </Step>
</Steps>

If no configuration file is found, auto-cpufreq will use default automatic settings.

## Quick Example

Here's a minimal configuration file example:

```ini
# Settings when connected to AC power
[charger]
governor = performance
turbo = auto

# Settings when on battery
[battery]
governor = powersave
turbo = auto
```

<Tip>
  After creating or modifying your config file, use `auto-cpufreq --monitor` to see how the settings will be applied without making permanent changes.
</Tip>
