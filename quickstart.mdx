This quickstart guide will get you from installation to monitoring your CPU optimizations in just a few minutes.

## Prerequisites

- **Linux system** with kernel 5.0+ (any distribution)
- **sudo/root access** for installation and CPU frequency control
- **Intel, AMD, or ARM CPU** (all architectures supported)

## Installation

<Steps>
  <Step title="Clone the repository">
    ```bash
    git clone https://github.com/AdnanHodzic/auto-cpufreq.git
    cd auto-cpufreq
    ```
    
    <Warning>
      Always use `git clone` rather than downloading a ZIP file. The project uses git-based versioning, and ZIP downloads will cause build errors.
    </Warning>
  </Step>
  
  <Step title="Run the installer">
    ```bash
    sudo ./auto-cpufreq-installer
    ```
    
    Follow the on-screen prompts and select option **i** to install.
    
    The installer will:
    - Install Python dependencies
    - Copy auto-cpufreq binaries to `/usr/local/bin/`
    - Set up the systemd service
    - Detect and disable conflicting services (TLP, GNOME Power Profiles, TuneD)
  </Step>
  
  <Step title="Verify installation">
    ```bash
    auto-cpufreq --version
    ```
    
    You should see the installed version number.
  </Step>
</Steps>

## Try Monitor Mode

Before installing the daemon, test auto-cpufreq in monitor mode to see what it would do:

```bash
sudo auto-cpufreq --monitor
```

This will show:
- Current CPU frequency and governor
- Battery state
- System load
- Temperature
- **Suggestions** for what auto-cpufreq would change (highlighted in yellow)

Press **Q** or **Ctrl+C** to exit. No changes are made in monitor mode.

## Install the Daemon

Once you're satisfied with the suggestions, install auto-cpufreq as a daemon for automatic, persistent optimizations:

```bash
sudo auto-cpufreq --install
```

This will:
- Enable the auto-cpufreq systemd service
- Start the daemon immediately
- Apply optimizations automatically based on battery state and system load

<Note>
  After installation, optimizations run continuously in the background. Your settings persist across reboots.
</Note>

## View Live Stats

Check what auto-cpufreq is doing in real-time:

```bash
auto-cpufreq --stats
```

The stats display shows:
- Current battery state and percentage
- Active CPU governor
- Current and maximum frequencies
- Turbo boost status
- System load and recent governor changes

<Tip>
  The stats command doesn't require `sudo` since it only reads data, not makes changes.
</Tip>

## Test Battery vs AC Behavior

To see auto-cpufreq in action:

<Tabs>
  <Tab title="On Battery">
    Unplug your laptop and check stats:
    
    ```bash
    auto-cpufreq --stats
    ```
    
    You should see:
    - Governor: **powersave**
    - Turbo boost: **Disabled**
    - Frequencies: Lower range
    
    This maximizes battery life.
  </Tab>
  
  <Tab title="On AC Power">
    Plug in your laptop and check stats again:
    
    ```bash
    auto-cpufreq --stats
    ```
    
    You should see:
    - Governor: **performance**
    - Turbo boost: **Enabled**
    - Frequencies: Higher range
    
    This maximizes performance.
  </Tab>
</Tabs>

The transition happens automatically within 1-2 seconds of plugging/unplugging.

## Common Post-Install Tasks

### Enable Bluetooth Management

If you want auto-cpufreq to disable Bluetooth on boot (you can still enable it manually):

```bash
sudo auto-cpufreq --bluetooth_boot_off
```

### Create a Configuration File

For custom settings that differ from defaults:

```bash
sudo nano /etc/auto-cpufreq.conf
```

Add your configuration (example):

```ini
[charger]
governor = performance
turbo = auto

[battery]
governor = powersave
turbo = never
```

Restart the daemon to apply changes:

```bash
sudo systemctl restart auto-cpufreq
```

See the [Config File](/configuration/config-file) guide for all available options.

### Check Daemon Status

Verify the daemon is running:

```bash
systemctl status auto-cpufreq
```

View recent logs:

```bash
journalctl -u auto-cpufreq -n 50
```

## Troubleshooting

<AccordionGroup>
  <Accordion title="Daemon won't start">
    Check for conflicting services:
    
    ```bash
    # GNOME Power Profiles
    sudo systemctl status power-profiles-daemon
    
    # TLP
    sudo systemctl status tlp
    
    # TuneD
    sudo systemctl status tuned
    ```
    
    Disable any that are running:
    
    ```bash
    sudo systemctl mask power-profiles-daemon
    sudo systemctl stop tlp && sudo systemctl disable tlp
    sudo systemctl stop tuned && sudo systemctl disable tuned
    ```
  </Accordion>
  
  <Accordion title="Governor not changing">
    Check if you have governor override set:
    
    ```bash
    # Reset any overrides
    sudo auto-cpufreq --force=reset
    sudo auto-cpufreq --turbo=auto
    ```
    
    Then restart the daemon:
    
    ```bash
    sudo systemctl restart auto-cpufreq
    ```
  </Accordion>
  
  <Accordion title="High CPU temperatures">
    You may need a more conservative configuration:
    
    ```ini
    [charger]
    governor = powersave
    turbo = never
    
    [battery]
    governor = powersave
    turbo = never
    ```
    
    Also check your laptop's cooling system (clean vents, repaste CPU if needed).
  </Accordion>
  
  <Accordion title="Stats not updating">
    Make sure the daemon is running:
    
    ```bash
    sudo systemctl status auto-cpufreq
    ```
    
    If it's not running:
    
    ```bash
    sudo auto-cpufreq --install
    ```
  </Accordion>
</AccordionGroup>

For more help, see the [Troubleshooting Guide](/guides/troubleshooting) or join the [Discord community](https://discord.gg/Sjauxtj6kH).

## Uninstalling

If you need to remove auto-cpufreq:

```bash
sudo auto-cpufreq --remove
```

This will:
- Stop and disable the daemon
- Remove the systemd service
- Restore your CPU to default settings
- Optionally remove the installed files

<Warning>
  If you installed via Snap or AUR, use the appropriate package manager to uninstall instead of `--remove`.
</Warning>
