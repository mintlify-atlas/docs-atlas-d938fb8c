The `--version` flag displays version information for your auto-cpufreq installation.

## Syntax

```bash
auto-cpufreq --version
```

## Description

The version command shows:
- Current auto-cpufreq version number
- Linux distribution information
- System details

This is useful for:
- Verifying installation
- Checking if updates are needed
- Reporting bugs (include version in reports)
- Confirming compatibility

## Requirements

- **No root required**: Can be run as normal user

## Example Usage

```bash
auto-cpufreq --version
```

## Expected Output

Typical version output:

```
-------------------------------- auto-cpufreq ---------------------------------

Linux distro: Ubuntu 22.04 LTS
Linux kernel: 5.15.0-58-generic

----- auto-cpufreq version -----

Version: 2.0.0

-------------------------------------------------------------------------------
```

## Output Components

### Distribution Information

```
Linux distro: Ubuntu 22.04 LTS
Linux kernel: 5.15.0-58-generic
```

Shows:
- Distribution name and version
- Kernel version

### Version Number

```
----- auto-cpufreq version -----

Version: 2.0.0
```

Displays the installed auto-cpufreq version.

## Use Cases

### Check Current Version

Before updating, check your current version:

```bash
auto-cpufreq --version
```

### Verify Installation

Confirm auto-cpufreq is installed:

```bash
auto-cpufreq --version
```

If installed, you'll see version info. If not:
```
command not found: auto-cpufreq
```

### Bug Reports

Include version when reporting bugs:

```bash
auto-cpufreq --version > version-info.txt
```

Attach to your GitHub issue.

### Compare Versions

Check if you're running the latest version:

1. Check installed version:
   ```bash
   auto-cpufreq --version
   ```

2. Check latest release:
   ```
   https://github.com/AdnanHodzic/auto-cpufreq/releases/latest
   ```

3. Update if needed:
   ```bash
   sudo auto-cpufreq --update
   ```

## Version Format

auto-cpufreq uses semantic versioning:

```
MAJOR.MINOR.PATCH
```

For example, version `2.0.0`:
- **MAJOR**: 2 - Major version (breaking changes)
- **MINOR**: 0 - Minor version (new features)
- **PATCH**: 0 - Patch version (bug fixes)

## Troubleshooting

### Command Not Found

```
auto-cpufreq: command not found
```

**Solutions:**

1. **Check if installed:**
   ```bash
   which auto-cpufreq
   ```

2. **Snap installation:**
   ```bash
   /snap/bin/auto-cpufreq --version
   ```

3. **Check PATH:**
   ```bash
   echo $PATH
   ```

4. **Reinstall if necessary:**
   ```bash
   sudo snap install auto-cpufreq  # For snap
   ```

### Version Seems Wrong

If version doesn't match expected:

1. **Check which binary is being used:**
   ```bash
   which auto-cpufreq
   ```

2. **Check for multiple installations:**
   ```bash
   find /usr -name auto-cpufreq 2>/dev/null
   find /opt -name auto-cpufreq 2>/dev/null
   ```

3. **Snap vs system installation:**
   ```bash
   /snap/bin/auto-cpufreq --version     # Snap version
   /usr/bin/auto-cpufreq --version      # System version
   ```
