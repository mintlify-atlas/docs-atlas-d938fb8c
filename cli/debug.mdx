---

The `--debug` flag displays comprehensive system and auto-cpufreq debug information, useful for troubleshooting issues and submitting bug reports.

## Syntax

```bash
sudo auto-cpufreq --debug
```

## Description

Debug mode displays:
- Distribution and kernel information
- Complete system hardware details
- CPU frequency driver and governor information
- Battery status and charging state
- Battery charge thresholds (if supported)
- Current CPU frequencies for all cores
- Temperature readings
- System load averages
- Turbo boost status
- auto-cpufreq version information
- Python environment details
- Application resource usage
- Configuration file information (if used)

This comprehensive output is essential for:
- Diagnosing issues
- Submitting bug reports
- Verifying system compatibility
- Understanding current system state

## Requirements

- **Root privileges**: Required to access all system information

## Example Usage

```bash
sudo auto-cpufreq --debug
```

### Save Debug Output to File

```bash
sudo auto-cpufreq --debug > auto-cpufreq-debug.txt
```

### Debug with Custom Config

```bash
sudo auto-cpufreq --debug --config=/etc/auto-cpufreq.conf
```

## Use Cases

### Troubleshooting Issues

When auto-cpufreq isn't working as expected:

```bash
sudo auto-cpufreq --debug > debug-output.txt
```

Review the output to identify:
- Unsupported hardware
- Driver issues
- Configuration problems
- System incompatibilities

### Submitting Bug Reports

When reporting bugs on GitHub:

```bash
sudo auto-cpufreq --debug > debug-$(date +%Y%m%d).txt
```

Attach this file to your bug report at:
https://github.com/AdnanHodzic/auto-cpufreq/issues

### Verifying Installation

Confirm auto-cpufreq is properly installed and configured:

```bash
sudo auto-cpufreq --debug
```

Check:
- Version is correct
- CPU driver is supported
- Battery is detected (if laptop)
- Turbo boost is available (if applicable)

### Comparing Before/After Changes

Capture state before making changes:

```bash
sudo auto-cpufreq --debug > before.txt
# Make changes
sudo auto-cpufreq --debug > after.txt
diff before.txt after.txt
```

### System Documentation

Document your system configuration:

```bash
sudo auto-cpufreq --debug > system-config-$(hostname)-$(date +%Y%m%d).txt
```

## Notes

- **Root required**: Some information requires root privileges
- **Static snapshot**: Debug output is a one-time snapshot, not continuous
- **Include in bug reports**: Always include debug output when reporting issues
- **No modifications**: Debug mode doesn't change any settings
- **Quick execution**: Completes in under a second

## Analyzing Debug Output

### Check CPU Driver

Look for:
```
Driver: intel_pstate
```

Or:
```
Driver: acpi-cpufreq
```

Both are supported. If showing something else, there may be compatibility issues.

### Check Governor Support

Cross-reference with available governors:
```bash
cat /sys/devices/system/cpu/cpu0/cpufreq/scaling_available_governors
```

### Check Battery Detection

Look for:
```
Battery is: charging
```

Or:
```
Battery is: discharging
```

If battery isn't detected on a laptop, there may be a driver issue.

### Check Turbo Support

Look for:
```
----- Turbo boost -----

Status: off
Available: yes
```

- **Available: yes** - Turbo boost is supported
- **Available: no** - CPU doesn't support turbo boost

When submitting a bug report, include:

1. **Description of the issue**
2. **Steps to reproduce**
3. **Expected behavior**
4. **Actual behavior**
5. **Debug output:**
   ```bash

   ```
6. **Installation method** (snap, AUR, git, etc.)
7. **Daemon status:**
   ```bash
   sudo systemctl status auto-cpufreq
   ```

## Troubleshooting

### Incomplete Debug Output

If debug output seems incomplete:

1. **Ensure root privileges:**
   ```bash
   sudo auto-cpufreq --debug
   ```

2. **Check for errors:**
   ```bash
   sudo auto-cpufreq --debug 2>&1 | tee debug-with-errors.txt
   ```

### Permission Denied

```
ERROR: Root privileges required for debug information
```

**Solution:**
```bash
sudo auto-cpufreq --debug
```

### Missing Information

If certain sections are missing:
- **Battery info**: Normal for desktop systems
- **Turbo boost**: Normal for CPUs without turbo
- **Thresholds**: Normal for systems without threshold support

## Privacy Considerations

Debug output includes:
- System manufacturer and model
- Hardware specifications
- Operating system version
- Installed software versions

**Does NOT include:**
- Personal files
- Passwords or credentials  
- Network information
- Running processes (beyond auto-cpufreq)

It's safe to share debug output publicly when reporting bugs.
