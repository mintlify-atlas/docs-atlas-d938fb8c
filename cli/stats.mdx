---

The `--stats` flag displays real-time statistics and information about CPU optimizations being applied by the running daemon.

## Syntax

```bash
auto-cpufreq --stats
```

## Description

The stats view shows:
- Current CPU frequencies for all cores
- Active CPU governor settings
- Battery status and charge level
- Temperature readings
- System load averages
- Applied optimizations
- Daemon activity and decisions

This provides visibility into what the daemon is doing in the background.

## Requirements

- **Running daemon**: The daemon must be installed and running
- **No root required**: Can be run as normal user (unlike --monitor)

## Example Usage

```bash
auto-cpufreq --stats
```

## Behavior

### On Startup

1. Checks if daemon is running
2. Detects configuration file usage
3. Checks for conflicts (GNOME Power Profiles, TLP)
4. Prompts for confirmation if conflicts detected
5. Starts displaying live statistics

### During Display

- Updates continuously in real-time
- Shows dynamic changes as daemon makes adjustments
- Reflects current system state
- Can run indefinitely

### Exiting

Press `Ctrl+C` to exit the stats view.

## Conflict Detection

If conflicting services are detected, you'll see a warning:

```
WARNING: GNOME Power Profiles daemon is running
This may conflict with auto-cpufreq

press Enter to continue or Ctrl + c to exit...
```

Press Enter to continue viewing stats, or Ctrl+C to exit.

## Use Cases

### Monitoring Daemon Activity

Watch what the daemon is doing in real-time:
```bash
auto-cpufreq --stats
```

Leave it running to observe:
- Governor changes when switching between AC/battery
- Frequency adjustments based on load
- Battery drain rate
- Temperature variations

### Verifying Installation

Confirm the daemon is working after installation:
```bash
auto-cpufreq --stats
```

Look for:
- Active governor settings
- Frequency scaling activity
- No error messages

### Troubleshooting Performance

Check if daemon is optimizing correctly:
```bash
auto-cpufreq --stats
```

Verify:
- Appropriate governor for power state (AC vs. battery)
- CPU frequencies are scaling properly
- Turbo boost behavior matches expectations

### Battery Life Analysis

Monitor battery stats while on battery power:
```bash
auto-cpufreq --stats
```

Observe:
- Battery discharge rate
- Power-saving governor is active
- CPU frequencies are reduced appropriately

## Notes

- **No root required**: Unlike --monitor and --live
- **Daemon must be running**: Shows error if daemon not installed
- **Real-time updates**: Displays current state continuously
- **Config file info**: Shows which config file is in use (if any)
- **Safe to run**: Read-only view, no system changes

## Error Messages

### Daemon Not Running

```
ERROR: auto-cpufreq daemon is not running

Run "sudo auto-cpufreq --install" to install daemon
```

**Solution:**
```bash
sudo auto-cpufreq --install
```

### Permission Issues

If you see permission errors, the daemon may not have been installed correctly:
```bash
sudo systemctl status auto-cpufreq
```

## Viewing Logs

For more detailed information, check daemon logs:

**systemd:**
```bash
sudo journalctl -u auto-cpufreq -f
```

**snap:**
```bash
sudo snap logs auto-cpufreq -f
```

## Configuration Information

If using a config file, stats will show:
```
Using settings defined in /etc/auto-cpufreq.conf file
```

This helps verify which configuration is active.
