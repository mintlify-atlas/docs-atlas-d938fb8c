---

Daemon mode runs auto-cpufreq as a background system service, providing continuous and automatic CPU optimization that persists across reboots.

## Installation

Install the auto-cpufreq daemon:

```bash
sudo auto-cpufreq --install
```

### What Happens During Installation

The installation process:

<Steps>
  <Step title="Pre-installation Checks">
    - Verifies the daemon is not already running
    - Checks for available CPU governors
    - Detects conflicting services (TLP, GNOME Power Profiles)
  </Step>
  <Step title="Script Deployment">
    - Installs `cpufreqctl.auto-cpufreq` control script to `/usr/local/bin/`
    - Deploys `auto-cpufreq-install` script
    - Deploys `auto-cpufreq-remove` script
  </Step>
  <Step title="Service Configuration">
    - Disables GNOME Power Profiles daemon (if detected)
    - Disables tuned service (if detected)
    - Creates stats file at `/var/run/auto-cpufreq.stats`
    - Optionally disables Bluetooth on boot
  </Step>
  <Step title="systemd Service Setup">
    - Creates and enables `auto-cpufreq.service`
    - Starts the daemon immediately
    - Configures automatic start on boot
  </Step>
</Steps>

### Installation Output

```bash
$ sudo auto-cpufreq --install

--------------------- Deploying auto-cpufreq as a daemon ----------------------

* Deploy auto-cpufreq install script

* Deploy auto-cpufreq remove script

GNOME Power Profiles Daemon is running.
auto-cpufreq will disable it to avoid conflicts.

* Disabling GNOME Power Profiles daemon

Created symlink /etc/systemd/system/auto-cpufreq.service → /usr/local/share/auto-cpufreq/auto-cpufreq.service

----------------- auto-cpufreq daemon installed and running ------------------

To view live stats, run:
auto-cpufreq --stats

auto-cpufreq makes all decisions automatically, if you would like to
configure certain settings to your own liking, please refer to:
https://github.com/AdnanHodzic/auto-cpufreq#configuring-auto-cpufreq

To disable and remove auto-cpufreq daemon, run:
sudo auto-cpufreq --remove

-------------------------------------------------------------------------------
```

<Check>
  The daemon is now running and will automatically start on every boot.
</Check>

## Managing the Daemon

### Using systemctl

Since auto-cpufreq runs as a systemd service, you can manage it with standard systemctl commands:

#### Check Status

```bash
sudo systemctl status auto-cpufreq
```

Output:
```text
● auto-cpufreq.service - auto-cpufreq - Automatic CPU speed & power optimizer
     Loaded: loaded (/usr/local/share/auto-cpufreq/auto-cpufreq.service; enabled; vendor preset: disabled)
     Active: active (running) since Mon 2024-03-13 09:15:23 UTC; 2h 45min ago
   Main PID: 1234 (python3)
      Tasks: 3 (limit: 19123)
     Memory: 15.2M
        CPU: 1min 23s
     CGroup: /system.slice/auto-cpufreq.service
             └─1234 /usr/bin/python3 /usr/local/bin/auto-cpufreq --daemon
```

#### Stop the Daemon

```bash
sudo systemctl stop auto-cpufreq
```

<Warning>
  Stopping the daemon stops CPU optimizations. Frequency scaling will revert to system defaults.
</Warning>

#### Start the Daemon

```bash
sudo systemctl start auto-cpufreq
```

#### Restart the Daemon

Useful after configuration changes:

```bash
sudo systemctl restart auto-cpufreq
```

#### Disable Auto-Start

Prevent the daemon from starting on boot (but keep it installed):

```bash
sudo systemctl disable auto-cpufreq
```

To re-enable:

```bash
sudo systemctl enable auto-cpufreq
```

### View Logs

Check daemon logs using journalctl:

```bash
sudo journalctl -u auto-cpufreq -f
```

View recent logs:

```bash
sudo journalctl -u auto-cpufreq -n 100
```

View logs from current boot:

```bash
sudo journalctl -u auto-cpufreq -b
```

auto-cpufreq --stats

## Daemon Behavior

### Optimization Cycle

The daemon operates in a continuous loop:

1. **Collect System Info**: Gathers CPU, temperature, and load data
2. **Detect Power Source**: Checks if on battery or AC power
3. **Apply Optimizations**: Sets appropriate governor and frequencies
4. **Manage Turbo**: Enables/disables turbo boost based on load
5. **Log Activity**: Writes to stats file
6. **Wait**: Sleeps for 2 seconds
7. **Repeat**: Goes back to step 1

The daemon automatically switches profiles when you plug in or unplug your laptop.

## Configuration File Integration

The daemon automatically uses the configuration file if present:

```bash
# Default config locations (checked in order):
# 1. /etc/auto-cpufreq.conf
# 2. ~/.config/auto-cpufreq/auto-cpufreq.conf
```

If a config file exists, you'll see:

```text
Using settings defined in /etc/auto-cpufreq.conf file
```

After modifying the configuration, restart the daemon:

```bash
sudo systemctl restart auto-cpufreq
```

<Tip>
  Test your configuration with `--live` mode before restarting the daemon to ensure it works as expected.
</Tip>

## Removing the Daemon

Uninstall the auto-cpufreq daemon:

```bash
sudo auto-cpufreq --remove
```

### What Happens During Removal

The removal process:

<Steps>
  <Step title="Service Cleanup">
    - Stops the auto-cpufreq service
    - Disables automatic start on boot
    - Removes the systemd service file
  </Step>
  <Step title="Service Restoration">
    - Re-enables GNOME Power Profiles daemon (if it was disabled)
    - Re-enables tuned service (if it was disabled)
    - Restores Bluetooth auto-start (if it was disabled)
  </Step>
  <Step title="File Cleanup">
    - Removes cpufreqctl control script
    - Removes install/remove scripts
    - Deletes stats file
    - Removes any override settings
  </Step>
</Steps>

### Removal Output

```bash
$ sudo auto-cpufreq --remove

--------------------- Removing auto-cpufreq daemon ---------------------------

Removing auto-cpufreq service...
Reloading systemd daemon...

Re-enabling GNOME Power Profiles daemon...

------------------------- auto-cpufreq daemon removed -------------------------

auto-cpufreq successfully removed.

-------------------------------------------------------------------------------
```

<Check>
  Your system returns to its pre-installation state with all original services restored.
</Check>

## Snap Package Installation

If you installed auto-cpufreq via Snap, the daemon commands work differently:

### Install Daemon (Snap)

```bash
sudo auto-cpufreq --install
```

This sets the snap daemon state to "enabled" using snapctl.

### Remove Daemon (Snap)

```bash
sudo auto-cpufreq --remove
```

This sets the snap daemon state to "disabled".

### Snap-Specific Paths

When using Snap:
- Stats file: `/var/snap/auto-cpufreq/current/auto-cpufreq.stats`
- Override state: `/var/snap/auto-cpufreq/current/override.pickle`
- Turbo override: `/var/snap/auto-cpufreq/current/turbo-override.pickle`

## Troubleshooting Daemon Issues

### Daemon Won't Start

Check for errors:

```bash
sudo systemctl status auto-cpufreq
sudo journalctl -u auto-cpufreq -n 50
```

Common issues:
- Conflicting services still running (TLP, laptop-mode-tools)
- Missing CPU governors
- Permission issues with stats file

### Stats File Growing Too Large

The daemon automatically truncates the stats file when it exceeds 10MB. If you want to manually clear it:

```bash
sudo systemctl stop auto-cpufreq
sudo rm /var/run/auto-cpufreq.stats
sudo systemctl start auto-cpufreq
```

### Daemon Not Responding to Power Changes

Restart the daemon:

```bash
sudo systemctl restart auto-cpufreq
```

Check if power supply detection is working:

```bash
ls -la /sys/class/power_supply/
```

### Configuration Not Applied

Verify the config file path:

```bash
auto-cpufreq --debug
```

Look for the line:
```text
Using settings defined in /etc/auto-cpufreq.conf file
```

If not shown, your config may be in the wrong location or have syntax errors.

## Best Practices

    Always run `--monitor` and `--live` modes first to ensure auto-cpufreq works correctly with your hardware.

  <Step title="Check Logs Regularly">

  </Step>
  <Step title="Update Configuration as Needed">

  </Step>
</Steps>
